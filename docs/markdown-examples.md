# Markdown Examples

This file shows short examples that follow the repository's Markdown rules.

## Top-level heading

Use one H1 per file. Subheadings use `##`, `###`, etc., and increase one level at a time.

## Lists

- Top-level list item
- Another item
    + Four-space indented sublist item

## Code block

```bash
echo "Hello, world"
```

Include the language after the opening fence for syntax highlighting.

## Links and images

- Inline link: [Lessons in AI](../README.md)
- Image with alt text:

  ![Example image](https://example.com/image.png)

## Emphasis and strong text

- Italic using underscores: _italic text_
- Strong using asterisks: **strong text**

## Spacing

- Place a blank line before and after fenced code blocks and before lists.
